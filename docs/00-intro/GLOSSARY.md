# GLOSSARY

## What
Project-wide glossary of core terms.

## Why
Reduces ambiguity across docs and crates.

## Non-goals
- Replacing detailed contract docs.

## Contracts
Each term links to its authoritative contract doc when available.

## Examples
- **run**: a single pipeline execution (see [../30-operations/RUN_ARTIFACTS.md](../30-operations/RUN_ARTIFACTS.md)).
- **plan**: an execution graph proposal (see [../10-architecture/CONTRACT_SPINE.md](../10-architecture/CONTRACT_SPINE.md)).
- **manifest**: run summary contract (see [../30-operations/RUN_ARTIFACTS.md](../30-operations/RUN_ARTIFACTS.md)).
- **record**: per-step execution record (see [../30-operations/RUN_ARTIFACTS.md](../30-operations/RUN_ARTIFACTS.md)).
- **report**: final bundle (see [../30-operations/REPORT_CONTRACT.md](../30-operations/REPORT_CONTRACT.md)).
- **pipeline**: preset configuration (see [../50-reference/PIPELINES.md](../50-reference/PIPELINES.md)).
- **profile**: named override set (see [../50-reference/PIPELINES.md](../50-reference/PIPELINES.md)).
- **stage**: contract-defined step (see [../10-architecture/CONTRACT_SPINE.md](../10-architecture/CONTRACT_SPINE.md)).
- **tool**: executable with identity (see [../10-architecture/CONTRACT_SPINE.md](../10-architecture/CONTRACT_SPINE.md)).
- **metric**: typed measurement (see [../10-architecture/CONTRACT_SPINE.md](../10-architecture/CONTRACT_SPINE.md)).
- **fixture**: deterministic test input (see [../40-policies/STYLE.md](../40-policies/STYLE.md)).

## Failure modes
If a term is used inconsistently, update this glossary first.

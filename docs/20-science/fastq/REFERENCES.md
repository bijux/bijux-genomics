# REFERENCES (FASTQ)

## What
Primary citations supporting FASTQ stages, metrics, and decisions.

## Why
References make tool assumptions explicit and reviewable.

## Non-goals
- A full literature survey.
- Replacing tool manuals.

## Contracts
- References must map to a stage, metric, or decision described in docs.

## Examples
### validate
- fastqvalidator: format validation tool

### trim/filter
- fastp: Chen et al. 2018

### screen
- fastq_screen: Wingett & Andrews 2018

## Failure modes
- Missing citations undermine reproducibility.

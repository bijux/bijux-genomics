# Bijux canonical FASTQ dataset (v1)

This folder contains the **frozen** FASTQ dataset used by Bijux for:

- image QA (functional correctness)
- benchmarks
- CI smoke checks

## What it is

A tiny, deterministic SE + PE dataset built specifically for Bijux. It includes:

- adapters appended to a subset of reads
- low‑quality tails
- short reads
- UMIs in the read header (`UMI:XXXX`)

The intent is to exercise trimming, filtering, QC, and UMI handling on realistic edge cases without downloading large public datasets.

## Files

- `BIJUX_SE_R1.fastq.gz`
- `BIJUX_PE_R1.fastq.gz`
- `BIJUX_PE_R2.fastq.gz`

## Source and provenance

These files are **synthetic** and generated by the Bijux tooling for deterministic QA/bench use.
They are not derived from external public datasets.

## Why it exists

We need a stable, reproducible input for:

- deterministic QA outcomes
- comparable benchmark results
- CI that does not depend on network downloads

## Stability contract

- Filenames and content are **immutable** for v1.
- Any changes require a version bump and updated checksums in
  `docs/contracts/fastq/canonical_dataset.md`.

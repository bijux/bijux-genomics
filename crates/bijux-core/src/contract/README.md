# Contract Map

This directory is the contract bible for Bijux. It defines all serialized interfaces and
compatibility rules.

## Layout
- `execution/`: execution graph, records, and runtime policies
- `run/`: run index, metadata, and run-scoped contracts
- `tooling/`: tooling contracts, selection, and constraints
- `version.rs`: contract versioning
- `canonical.rs`: canonical JSON + hashing rules for truth artifacts

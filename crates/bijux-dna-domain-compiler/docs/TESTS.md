# bijux-dna-domain-compiler Tests

The test suite protects compiler boundaries, generated-output determinism, planned-tool behavior,
and repository guardrails.

## Test Map

| Surface | Test file | Contract |
| --- | --- | --- |
| Workspace guardrails | `tests/guardrails.rs` | Applies `bijux-dna-policies` to the crate. |
| Generated determinism | `tests/determinism_generated_outputs.rs` | Repeated compiles produce identical generated bytes. |
| Planned-tool registry boundary | `tests/planned_tool_registry_boundaries.rs` | Planned-only FASTQ alternatives stay out of governed runtime registry entries while remaining visible in stage metadata. |
| Architecture boundaries | `tests/boundaries.rs` and `tests/boundaries/*.rs` | Locks docs layout, command inventory, dependency shape, and source/test tree intent. |

## Commands

```bash
CARGO_TARGET_DIR=artifacts/cargo-target cargo test -p bijux-dna-domain-compiler --no-default-features
CARGO_TARGET_DIR=artifacts/cargo-target cargo test -p bijux-dna-domain-compiler --no-default-features --test boundaries
CARGO_TARGET_DIR=artifacts/cargo-target cargo test -p bijux-dna-domain-compiler --no-default-features --test guardrails
CARGO_TARGET_DIR=artifacts/cargo-target cargo test -p bijux-dna-domain-compiler --no-default-features --test determinism_generated_outputs
CARGO_TARGET_DIR=artifacts/cargo-target cargo test -p bijux-dna-domain-compiler --no-default-features --test planned_tool_registry_boundaries
CARGO_TARGET_DIR=artifacts/cargo-target cargo clippy -p bijux-dna-domain-compiler --all-targets --no-default-features -- -D warnings
```

## Artifact Discipline

Tests that generate compiler output should place run directories beneath
`artifacts/domain-compiler-test-runs/` through the shared test support helper. Rust build products
should use `CARGO_TARGET_DIR=artifacts/cargo-target`.

## Failure Modes Covered

- Markdown outside root `README.md` or `docs/`.
- Docs allowance drift beyond ten docs.
- Binary commands that are not listed in [COMMANDS.md](COMMANDS.md).
- Forbidden execution-layer dependencies.
- Generated config drift and planned-tool promotion into governed runtime registries.

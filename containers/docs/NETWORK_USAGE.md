# Container Network Usage

Purpose: disclose where network access is required during container build/smoke workflows.

Policy:
- Build recipes that fetch remote content must be traceable through pinned sources/checksums.
- Network usage inventory is generated by:
  - `scripts/containers/generate-network-usage.sh`
- CI/lint gate:
  - `scripts/containers/check-network-disclosure.sh`

Offline mode:
- Set `BIJUX_OFFLINE=1` to enforce offline constraints in build/smoke scripts.
- Docker builds use `--network=none` in offline mode.
- Apptainer build paths are blocked in offline mode when remote fetch commands are present.

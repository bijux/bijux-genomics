# All-Domain Readiness Proof

This directory stores tracked deterministic readiness proof that spans the governed FASTQ, BAM,
and VCF benchmark surface as one all-domain contract.

`active-stage-catalog.tsv` is regenerated by
`bijux-dna bench readiness render-all-domain-active-stage-catalog` and keeps one governed row per
active FASTQ, BAM, and VCF stage, including active tool coverage, benchmark-ready coverage,
parser coverage, normalized schema presence, and report section coverage. Active scope excludes
`planned`, `future`, and equivalent not-yet-active lifecycle rows.

`adapter-coverage.tsv` is regenerated by
`bijux-dna bench readiness render-all-domain-adapter-coverage` and keeps one governed row per
active FASTQ, BAM, and VCF binding, proving the final job-bearing active scope retains
executable rendered-command coverage for every active row.

`expected-result-coverage.tsv` is regenerated by
`bijux-dna bench readiness render-all-domain-expected-result-coverage` and keeps one governed row
per active FASTQ, BAM, and VCF binding, proving the final job-bearing active scope retains one
canonical expected-result row for every active stage-tool-corpus-asset binding.

`active-scope-blockers.tsv` is regenerated by
`bijux-dna bench readiness render-all-domain-active-scope-blockers` and keeps one governed row
per candidate FASTQ, BAM, and VCF binding that is intentionally outside the final job-bearing
active scope, with the exact blocker type and blocker proof path for each excluded binding.

`ACTIVE_SCOPE_COMPLETE.json` is regenerated by
`bijux-dna bench readiness render-all-domain-active-scope-complete` and proves the governed
final all-domain active scope is non-ambiguous. The gate aggregates the retained-tool, active
stage catalog, active matrix, active-scope reduction, removed-scope, alias, parser, adapter,
output-contract, expected-result, report-map, local-job, blocker, placeholder-command, and fast
active-scope validator surfaces, and it keeps dedicated VCF imputation-identity plus VCF
postprocess-closure checks explicit with durable proof paths.

`local-job-coverage.tsv` is regenerated by
`bijux-dna bench readiness render-all-domain-local-job-coverage` and keeps one governed row per
active FASTQ, BAM, and VCF binding, proving the final job-bearing active scope retains one
executable local benchmark job row in the governed all-domain shell-script and argv outputs for
every active result binding.

`no-placeholder-command-check.json` is regenerated by
`bijux-dna bench readiness render-all-domain-no-placeholder-command-check` and proves the final
job-bearing active scope contains no template-style command execution drift such as `todo`,
`placeholder`, echo-only shells, unconditional success stubs, empty executables, or missing real
tool invocation.

`report-map-coverage.tsv` is regenerated by
`bijux-dna bench readiness render-all-domain-report-map-coverage` and keeps one governed row per
active FASTQ, BAM, and VCF binding, proving the final job-bearing active scope appears in a
governed report section and summary table for every active result binding.

`output-contract-coverage.tsv` is regenerated by
`bijux-dna bench readiness render-all-domain-output-contract-coverage` and keeps one governed row
per active FASTQ, BAM, and VCF binding, proving the final job-bearing active scope retains raw
outputs, normalized metrics, logs, manifest, and required index declarations from the domain-owned
output-contract surfaces.

`parser-fixture-coverage.tsv` is regenerated by
`bijux-dna bench readiness render-all-domain-parser-fixture-coverage` and keeps one governed row
per active FASTQ, BAM, and VCF binding, proving the final job-bearing active scope retains parser
fixture evidence for every active row.

`active-stage-tool-matrix.tsv` is regenerated by
`bijux-dna bench readiness render-all-domain-active-stage-tool-matrix` and keeps one governed row
per active FASTQ, BAM, and VCF stage-tool binding, including corpus, asset, adapter, parser,
schema, and unified lifecycle status coverage. Active scope excludes `planned`, `future`, and
equivalent not-yet-active lifecycle rows, excludes declaration-only rows without executable
adapter coverage, and excludes `not_benchmark_ready` rows that cannot remain in job-bearing active
scope.

`stage-tool-alias-check.json` is regenerated by
`bijux-dna bench readiness render-stage-tool-alias-check` and proves migration-only stage aliases
and separator-folded tool aliases do not leak into candidate, active, expected-result, or
rendered-command all-domain benchmark surfaces.

`no-declared-only-rows.json` is regenerated by
`bijux-dna bench readiness render-all-domain-no-declared-only-rows` and proves the governed
active all-domain scope contains no declaration-only rows without executable adapters.

`no-not-benchmark-ready-rows.json` is regenerated by
`bijux-dna bench readiness render-all-domain-no-not-benchmark-ready-rows` and proves the
governed active all-domain scope contains no `not_benchmark_ready` rows after executable-active
rows are reduced to job-bearing benchmark-ready scope.

`retained-tools.tsv` is regenerated by
`bijux-dna bench readiness render-all-domain-retained-tools` and keeps one retained row per active
tool across the unified stage-tool inventory after planned-only bindings are removed from active
scope, declaration-only rows are excluded from active eligibility, and `not_benchmark_ready` rows
are removed from job-bearing active scope.

Bindings that are intentionally outside that final active scope are tracked separately in
`../removed-from-scope.tsv`, regenerated by
`bijux-dna bench readiness render-removed-from-scope`.

`no-planned-rows.json` is regenerated by
`bijux-dna bench readiness render-all-domain-no-planned-rows` and proves the governed active
all-domain scope contains no `planned`, `future`, or equivalent not-yet-active lifecycle rows.
